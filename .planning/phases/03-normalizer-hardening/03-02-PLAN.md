    ```typescript
    // Leet speak character map -- security-focused minimal set (per D-07)
    // Maps digit/symbol substitutions commonly used in injection attacks
    const LEET_MAP: Record<string, string> = {
      '0': 'o',
      '1': 'i',
      '3': 'e',
      '4': 'a',
      '5': 's',
      '7': 't',
      '@': 'a',
      '$': 's',
      '!': 'i',
    };

    const LEET_RE = new RegExp('[' + Object.keys(LEET_MAP).map(k => k.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')).join('') + ']', 'g');
    ```

    Add the `decodeLeetSpeak()` function following the `decodeEncodings()` pattern (per D-08, D-12):

    ```typescript
    export function decodeLeetSpeak(input: string): DecodedInput[] {
      if (!input) return [];

      const decoded = input.replace(LEET_RE, (ch) => LEET_MAP[ch] ?? ch);

      // If nothing changed, no leet characters were present
      if (decoded === input) return [];

      return [{
        encoding: 'leet',
        decoded,
        originalOffset: 0,
        originalLength: input.length,
      }];
    }
    ```

    **Step 4: Run tests -- they must PASS (GREEN).**

    **Key design decisions:**
    - The function operates on already-normalized text (output of `normalize().text`), so zero-width chars and homoglyphs are already resolved.
    - Uses simple presence check: if `decoded === input`, no leet chars were found, return empty array. This is the simple heuristic recommended in RESEARCH.md Open Question 3.
    - The LEET_RE regex needs proper escaping for `$` and `!` characters in the character class. The `$` and `!` are literal inside a character class `[...]` so no escaping is strictly needed, but the escaping function handles edge cases.
    - This function is NOT called inside `normalize()` per D-08 and D-12. It is a standalone exported function.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Tampering | normalizer.ts decodeLeetSpeak() | mitigate | Separate variant decoder with 9-entry LEET_MAP; patterns run against decoded variant to catch leet-obfuscated injections (D-08) |
| T-03-08 | Information Disclosure | index.ts confidence boost | accept | +0.1 boost (lower than +0.15 for encoding) is conservative; leet-decoded matches are clearly labeled in description with '[leet decoded]' prefix for audit trail |
| T-03-09 | Denial of Service | normalizer.ts LEET_RE | accept | Simple character class regex with no nesting; O(n) guaranteed. 9-entry map is trivially fast. |
| T-03-10 | Tampering | Combined multi-vector evasion | mitigate | Pipeline applies all normalizations before scanning: NFKC -> zero-width strip -> homoglyph -> markdown -> HTML -> whitespace -> trim, then leet variant scanned separately. Multi-vector attacks are decomposed layer by layer. |
